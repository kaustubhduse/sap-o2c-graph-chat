## Decision-making notes
### Decision 1: Query-mode NodeDetail
- Added mode concept where NodeDetail can show multiple nodes linked to query output.
- Preserved single-node mode for manual graph clicks.
- Why: supports both exploratory and query-driven workflows.

### Decision 2: SQL-to-UI coupling
- New SQL assistant messages should auto-open NodeDetail in query mode.
- Why: improves explainability and reduces user clicks.

### Decision 3: Temporary black path emphasis
- Path highlights should be scoped to query context lifecycle.
- Why: avoids persistent visual clutter.

### Decision 4: Backend location normalization
- Keep canonical NL-to-SQL under `o2c-app/backend/nl-to-sql`.
- Why: deployment simplification and import determinism.

### Decision 5: URL-first DB config
- Prefer `MYSQL_URL` with fallback to component vars.
- Why: managed DB providers typically expose DSN URL first.

### Decision 6: Streaming compatibility
- Keep both standard and SSE streaming endpoints.
- Why: allows gradual rollout with fallback path.
